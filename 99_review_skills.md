# 99% Review Skills

A structured skill set defining the quality‑control criteria required for a 99% Review across four primary domains: **Content Strategy**, **Creative**, **Inclusive Design**, and **Pulse**.  
Each skill includes an ID and a clear description to support deterministic evaluation by AI agents.

---

## 1. Content Strategy

### 821 — Factual Accuracy
Content must be factually correct and up‑to‑date.

### 814 — Completeness
All sections, components, and required elements must be accounted for.

### 868 — Capitalization Standards
Capitalization must follow the defined style guide consistently.

### 873 — Terminology Clarity
Industry‑specific terminology must be clarified, defined, and used intentionally.

### 865 — Brand Alignment
Content must align with the brand’s identity, tone, and values.

### 867 — Human Readability
Content must read naturally and should not require rewriting to sound human.

### 871 — CTA Helpfulness
Calls‑to‑action must clearly guide users toward the intended next step.

### 866 — Authenticity & Engagement
Content must be engaging, relatable, and authentic.

### 872 — Lexicon Consistency
Correct terms and lexicon must be used consistently throughout the material.

### 889 — Content Strategy: Unclassified Standard
Standards not otherwise captured.

---

## 2. Creative

### 813 — Visual Media Compliance
Follows visual media guidelines for photography, illustration, and iconography.

### 806 — Color Compliance
Follows defined color usage standards.

### 820 — Typography Compliance
Typography (size, style, hierarchy) must comply with the design style guide.

### 854 — Motion & Interaction Compliance
Motion and interaction patterns follow established standards.

### 852 — Card Styling Compliance
Card‑based components follow layout and styling rules.

### 853 — Spacing Compliance
Spacing rules follow design system spacing tokens or guidelines.

### 879 — Creative: Unclassified Standard
Standards not otherwise captured.

---

## 3. Inclusive Design

### 815 — Inclusive Design Guidance Applied
All previously identified inclusive‑design requirements or recommendations must be included.

### 822 — Annotation Inclusion
Annotations necessary for inclusive understanding and implementation must be included.

### 808 — Accessibility Risk Identification
Accessibility risks must be explicitly identified and documented.

### 882 — Impact Documentation
Groups potentially negatively impacted must be documented.

### 892 — Inclusive Design: Unclassified Standard
Standards not otherwise captured.

---

## 4. Pulse (Design System / Figma)

### 823 — Component Definition Quality
New components or augmentation requests must be clearly defined and annotated for handoff.

### 816 — Functional Appropriateness
Component use case must be appropriate, consistent, and meet expected functionality.

### 809 — Library Linkage
Pulse components/resources must link to correct current versions.

### 840 — Token Application
Design tokens must be assigned to all available values.

### 885 — Pulse: Unclassified Standard
Standards not otherwise captured.

### 895 — Figma Naming Standards
File pages and Figma section naming must follow 99% Review standards.
